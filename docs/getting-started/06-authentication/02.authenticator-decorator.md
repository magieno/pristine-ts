# The `@authenticator` decorator

`@authenticator()` is how you tell a route (or every route in a controller) which
authentication strategy applies. It runs before the route handler — if authentication
fails, the handler never executes and the framework returns a 401.

> **You'll learn**
> - How to apply `@authenticator()` at the controller and route level
> - How decorator order works when both are present
> - How to write your own authenticator

## Method-level: protect one route

```ts
import {authenticator} from "@pristine-ts/security";
import {JwtAuthenticator} from "@pristine-ts/jwt";

@injectable()
@controller("/products")
export class ProductsController {
  @route(HttpMethod.Get, "")
  list() {
    // public — anyone can hit this
    return this.productService.list();
  }

  @authenticator(JwtAuthenticator)
  @route(HttpMethod.Delete, "/{id}")
  delete(@routeParameter("id") id: string) {
    // requires a valid JWT — see jwt.md
    return this.productService.delete(id);
  }
}
```

The `@authenticator()` decorator wraps the single method below it. `list` stays public;
`delete` requires authentication.

## Class-level: protect every route in a controller

```ts
@authenticator(JwtAuthenticator)
@injectable()
@controller("/admin")
export class AdminController {
  @route(HttpMethod.Get, "/dashboard")
  dashboard() {
    return this.adminService.dashboard();
  }

  @route(HttpMethod.Post, "/users")
  createUser(@body() options: CreateUserOptions) {
    return this.adminService.createUser(options);
  }
}
```

Every route in `AdminController` now requires JWT authentication. Cleaner than annotating
each method when the entire controller is private.

## Method-level overrides

Method-level `@authenticator` overrides the class-level one for that method:

```ts
@authenticator(JwtAuthenticator)
@injectable()
@controller("/api")
export class ApiController {
  @route(HttpMethod.Get, "/private")
  privateEndpoint() {
    // JWT required (inherited from class)
  }

  @authenticator(NoAuthAuthenticator)  // override — make this one public
  @route(HttpMethod.Get, "/public")
  publicEndpoint() {
    return {ok: true};
  }
}
```

This is occasionally useful for "everything's private except this one health-check
endpoint" patterns.

## Writing your own authenticator

Implement `AuthenticatorInterface`:

```ts
import {AuthenticatorInterface} from "@pristine-ts/security";
import {IdentityInterface, Request} from "@pristine-ts/common";

export class ApiKeyAuthenticator implements AuthenticatorInterface {
  private context: any = {};

  setContext(context: any): this {
    this.context = context;
    return this;
  }

  async authenticate(request: Request): Promise<IdentityInterface> {
    const apiKey = request.getHeader("x-api-key");
    if (apiKey === undefined) {
      throw new Error("Missing x-api-key header");
    }
    const userId = await this.lookupUserByApiKey(apiKey);
    if (userId === undefined) {
      throw new Error("Invalid API key");
    }
    return {
      id: userId,
      claims: {authMethod: "api-key"},
    };
  }

  private async lookupUserByApiKey(key: string): Promise<string | undefined> {
    // your lookup logic
  }
}
```

Two methods:
- **`setContext(context)`** — the framework calls this with the options object passed to
  `@authenticator(Klass, options)`. Use it for per-route configuration ("which scope is
  required for this route?").
- **`authenticate(request)`** — receives the Pristine `Request`, returns an
  `IdentityInterface` on success, throws on failure.

Pass the class, not an instance, to `@authenticator()`. The framework constructs an
instance per request:

```ts
@authenticator(ApiKeyAuthenticator)
@route(HttpMethod.Get, "/secret")
getSecret() { /* ... */ }
```

If you want to pass options, the framework also accepts an instance:

```ts
@authenticator(new ApiKeyAuthenticator(), {requireScope: "admin"})
```

## The flow, end-to-end

1. Request comes in.
2. Networking module identifies the matched route.
3. If the route (or its controller) has `@authenticator(Klass)`, the framework
   instantiates `Klass`, calls `setContext(options)`, and `authenticate(request)`.
4. If `authenticate` returns an identity, the IdentityProvider runs (`provide(identity)`)
   to enrich it.
5. The enriched identity is attached to the request context and passed to the route
   handler (and any voters that ask).
6. If `authenticate` throws, the framework returns 401 and the handler never runs.

## Pitfalls

- **The class form vs instance form difference**: passing the class lets the framework
  construct + DI-inject a fresh instance per request. Passing an instance reuses the
  same object across requests — fine for stateless authenticators, **bad if your
  authenticator stores per-request state on `this`**.
- **`setContext` is called before `authenticate`**, but only once per request. Don't
  expect it to be called repeatedly with different contexts.
- **Throwing means 401, not 500.** If your authenticator wants to surface "service
  unavailable" specifically, throw a typed error and have an HttpErrorResponseInterceptor
  rewrite the status code.
