# Telemetry

Pristine ships with a built-in tracing system: every request is wrapped in a **trace**,
and each trace contains a tree of **spans** describing where time was spent. Spans nest
naturally — the framework auto-spans the request lifecycle, and your code adds its own
spans for the parts that matter.

Three things make tracing useful in practice: low ergonomics overhead in your code, a
clean default output you can read without a backend, and pluggable transports for when
you do want to ship to one (X-Ray, your APM, etc.).

> **You'll learn**
> - The trace + span data model and the request lifecycle Pristine auto-instruments.
> - Two ways to add spans to your code: the `TracingManager` API and the `SpanRunner` helper.
> - Two built-in tracers — `ConsoleTracer` (tree to stdout) and `FileTracer` (per-trace files).
> - How to write a custom tracer, and how to disable AWS X-Ray without removing the package.
> - Crash isolation: a misbehaving tracer can never take down the app.
